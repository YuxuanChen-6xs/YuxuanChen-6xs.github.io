## 常见报错与修复方案（必看）

即使有千聚这样的平台，刚上手时也会遇到几个高频报错。我全踩了一遍，直接把解决方案写在这里，你能少走很多弯路。

### 错误一：`401 Authentication Error`

- **原因**：API Key 无效、过期，或者格式错误。
- **修复方法**：去千聚后台检查你的 API Key 是否已启用，重新复制粘贴，请确保不包含多余空格。

### 错误二：`404 Not Found` 或 `Model Not Found`

- **原因**：模型名称写错了，或者该模型在千聚平台暂未上线。
- **修复方法**：在千聚控制台查看模型列表，确认 `gpt-5.2-codex` 名称完全一致。注意大小写和连字符。

### 错误三：`Rate Limit Exceeded` (429)

- **原因**：请求频率过高，超出单密钥的并发限制。
- **修复方法**：降低请求频率，或使用千聚的并发通道（不分组的默认渠道比限时特价分组并发量更高）。如果需要更高并发，可以考虑升级分组。

### 错误四：连接超时或 `Connection Error`

- **原因**：网络不稳定，或你的 DNS 解析不到千聚服务器。
- **修复方法**：确认 `base_url` 写的确实是 `https://www.qianjuai.com/v1`，注意是 `https`。如果仍然超时，先手动访问网站看能否打开，再尝试重启网络。
