## 为什么 429 总是缠着你？

要解决 429，首先得明白它的本质。`429` 状态码意味着你触发了服务端的**速率限制（Rate Limit）**。官方 API 为了保护自身基础设施，会为每个 API Key 设定一个单位时间内的最大请求次数（RPM）和最大 Token 吞吐量（TPM）。

如果你直接调用 OpenAI 或 Claude 的原生 API，你面临的是最严格的速率限制。尤其是那些刚注册的、或者充了几十刀的入门级账户，RPM 可能只有可怜的 3-5，一旦你并发高了，或者单纯在轮询任务中多跑了几次，`429` 就会立刻弹出来。

传统的解决方案是什么？
1.  **买更贵的套餐**：充更多钱，购买更高的速率上限（但成本直线上升）。
2.  **自己搞代理池**：多注册几个账号，用负载均衡轮询。这很麻烦，而且有封号风险。
3.  **写重试逻辑**：代码里加 `time.sleep()` 和指数退避。能解决问题，但严重拖慢开发效率。

这些方法要么要钱，要么要命，要么两者都要。

而我要说的“隐藏低价选项”，恰恰绕过了这个矛盾的核心。
