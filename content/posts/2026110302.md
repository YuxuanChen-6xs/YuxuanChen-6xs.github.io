## 02 | 核心接入：Node.js 示例

所有的麻烦都来自一个源头：**配置纠结**。

无论是用 `axios` 还是 `openai` 官方库，最关键的一步，就是改 `baseURL`。

### 2.1 完整代码（推荐使用 openai 库）

先安装依赖（如果在企业内网，建议锁定版本）：

bash
npm install openai@^4.0.0 dotenv


然后在项目根目录新建 `.env` 文件：

env
# 千聚 API Key（在您的千聚面板中获取）
QIANJU_API_KEY=sk-你的key
# 千聚 API 基础地址
QIANJU_BASE_URL=https://www.qianjuai.com/v1
# 使用的模型
MODEL_NAME=deepseek-chat


接着创建 `deepseek-proxy.js`：

javascript
import OpenAI from 'openai';
import dotenv from 'dotenv';

dotenv.config();

// 初始化客户端——关键一步：改 baseURL
const client = new OpenAI({
  baseURL: process.env.QIANJU_BASE_URL,
  apiKey: process.env.QIANJU_API_KEY,
});

/**
 * 调用 DeepSeek V3.2 对话接口
 * @param {string} userMessage - 用户输入
 * @returns {Promise<string>} 模型回复
 */
async function chatWithDeepSeek(userMessage) {
  try {
    const response = await client.chat.completions.create({
      model: process.env.MODEL_NAME,
      messages: [
        { role: 'system', content: '你是一个专业的企业级Node.js助手。' },
        { role: 'user', content: userMessage },
      ],
      // 企业场景推荐开启流式输出
      // stream: true,
      max_tokens: 2048,
      temperature: 0.7,
    });

    return response.choices[0]?.message?.content || '';
  } catch (error) {
    console.error('API调用失败:', error.message);
    // 建议抛出或返回兜底文案
    throw new Error(`DeepSeek 调用失败: ${error.message}`);
  }
}

// 测试
(async () => {
  const reply = await chatWithDeepSeek('用 Node.js 写一个简单的 HTTP 服务器');
  console.log(reply);
})();


> **关键提醒**：`baseURL` 一定要设置成 `https://www.qianjuai.com/v1`。这个地方写错了，神仙也救不了。

**可能遇到的坑：**

1. **Error 401** → API Key 不对，去面板重新复制。
2. **Error 404** → `baseURL` 写成了 `https://www.qianjuai.com/` 忘加 `/v1`。
3. **ETIMEOUT** → 内网限制了 HTTPS 出站，需要配置防火墙允许 `*.qianjuai.com`。
4. **rate limit** → 默认分组并发较高，建议逐渐增加请求量；企业级可联系千聚开白名单。
