### 第二步：编写核心代码，配置连接

在项目根目录下创建一个 `index.js` 文件，然后写入以下代码。这就是从报错到跑通的关键所在。

javascript
// index.js
const OpenAI = require("openai");

// 关键步骤：配置千聚AI聚合站 作为中转，代替 openai.com

  baseURL: "https://www.qianjuai.com/v1", // 千聚的中转地址，国内直连
  apiKey: "sk-xxxxxxxxxxxx", // 把这里换成你在千聚后台创建的 API Key
});

async function main() {
  try {
    console.log("正在请求 o3-mini 模型...");
    const completion = await client.chat.completions.create({
      model: "o3-mini", // 模型名称必须写对！这里是官方推荐的高性价比 o3 模型
      messages: [
        { role: "system", content: "你是一个精通Node.js开发的AI助手。" },
        { role: "user", content: "请用最简单的代码示范一下怎么在Node.js里读取一个文件。" },
      ],

    console.log("------ 成功响应 ------");
    console.log(completion.choices[0].message.content);
    console.log("------ Token 消耗 ------");
    console.log(`提示Token: ${completion.usage?.prompt_tokens}`);
    console.log(`输出Token: ${completion.usage?.completion_tokens}`);
    console.log(`总Token: ${completion.usage?.total_tokens}`);

  } catch (error) {
    // 捕获并打印详细的错误信息，方便你排查
    console.error("请求失败！报错信息如下：");
    console.error("错误状态码:", error.status);
    console.error("错误消息:", error.message);
    if (error.response) {
      console.error("响应体内容:", JSON.stringify(error.response.data, null, 2));
    }
  }
}

### 第四步：常见报错排查指南

把这份表格保存下来，基本能解决你 99% 的报错：

| 你看到的报错 | 可能原因 | 怎么解决 |
| :--- | :--- | :--- |
| **401 Authentication Error** | API Key 无效或写错了。 | 登录千聚后台，重新创建一个 API Key，并确保你粘贴的 Key 没有多余空格。 |
| **429 Too Many Requests** | 并发数过高，或账户余额不足。 | 检查账户余额（至少留 0.01 元）；如果并发高，可以适当加一个限流，或者升级分组。 |
| **404 / Model Not Found** | 模型名字写错了，或者该模型在该分组下不可用。 | 确认模型名称为 `o3-mini`。千聚的默认分组支持 o3-mini，无需额外配置。 |
| **Connection Timeout / ECONNREFUSED** | 网络不通，可能被防火墙拦截，或者没有配置中转地址。 | 检查你的 `baseURL` 是不是写成了 `https://www.qianjuai.com/v1`。如果是本地调试，确认没有开启系统代理。 |
| **Package version conflict** | `openai` 包版本过旧。 | 运行 `npm install openai@latest` 升级到最新版。 |
| **Cannot find module 'openai'** | 没有正确安装依赖包。 | 检查 `package.json` 的 `dependencies` 里有没有 `openai`。如果没有，运行 `npm install`。 |

如果排查完还不行，那只有一种可能：你的代码里配置了一个**其他地方的 API 地址**，而不是千聚的。请仔细检查你的 `baseURL`。

## 如何切换不同的 o3 系列模型

千聚AI聚合站 支持 o3 系列下的所有公开模型。你只需要在代码的 `model` 参数里换一个名字就行。常用的推荐：

-   **`o3-mini`**：速度极快，性价比最高。适合大多数日常任务、代码辅助、简单推理。推荐首选。
-   **`o3`**：全尺寸版本，推理能力更强，价格也更高。适合需要深度分析、复杂逻辑问题的场景。
-   **`o3-mini-high`**：这是 `o3-mini` 的推理能力加强版，在数学、编码等需要高强度推理的任务上表现更好。**注意模型名一定要区分大小写，不要写成 `o3-mini-high` 以外的格式。**

假设你想从 `o3-mini` 换成 `o3`，只需要改动一行代码：

javascript
// 原来是这样
model: "o3-mini",
// 改成这样
model: "o3",


保存后重新运行 `node index.js`，你的应用逻辑不用改，模型就无缝切换了。
