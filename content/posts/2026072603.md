## 坑都踩过了：Node.js接入常见避坑指南

光给代码不够，我在接入过程中踩了几个坑，在这里一并列出来，你遇到了不用慌。

**坑1：环境变量没设对，API Key泄露**
我之前为了测试，直接把API Key写死在代码里，结果一不留神push到了公开仓库。
**避坑**：用 `.env` 文件，把Key藏在环境变量里。

bash
// .env 文件
 QIANJU_API_KEY=你的千聚API Key


然后在代码里：

javascript
const openai = new OpenAI({
   apiKey: process.env.QIANJU_API_KEY,
   baseURL: 'https://www.qianjuai.com/v1',
});


**坑2：模型选择错误，代码抛异常**
千聚支持的模型很多，但不同分组费率不同。如果你在“默认分组”里选了“Claude 3 Opus”，可能不在支持范围内。
**避坑**：官网有清晰的分组说明。一般用“默认分组”就好，覆盖了OpenAI、Claude、Gemini等主流模型。

**坑3：并发100+请求，结果超时**
我之前跑批量任务，一次发了200个并发请求，结果有几条卡住了。
**避坑**：千聚的API本身**无并发限制**，但你自己的网络环境可能有瓶颈。建议在客户端做一下并发控制，比如使用 `p-limit` 库。

javascript
import pLimit from 'p-limit';
 const limit = pLimit(10); // 每次只发10个并发

 const tasks = yourArray.map(item => {
   return limit(() => openai.chat.completions.create({...}));
 });
 await Promise.all(tasks);
