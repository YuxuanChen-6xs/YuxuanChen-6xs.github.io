## 手把手搭Node.js脚手架

这个才是今天的核心操作。以`express`框架为例，我们搭一个极简的“模型一键盘”：

javascript
const express = require('express');
const OpenAI = require('openai');

const app = express();
app.use(express.json());

// 定义模型快速切换配置
const modelConfig = {
  ernie: { model: 'ernie-4.0-8k-latest', baseURL: 'https://www.qianjuai.com/v1' },
  qwen: { model: 'qwen-turbo', baseURL: 'https://www.qianjuai.com/v1' },
  deepseek: { model: 'deepseek-chat', baseURL: 'https://www.qianjuai.com/v1' },
  gpt4: { model: 'gpt-4o', baseURL: 'https://www.qianjuai.com/v1' },
  gemini: { model: 'gemini-2.0-flash-exp', baseURL: 'https://www.qianjuai.com/v1' },
};

app.post('/chat', async (req, res) => {
  const { provider = 'ernie', message } = req.body;
  const cfg = modelConfig[provider] || modelConfig.ernie;

  const client = new OpenAI({
    apiKey: process.env.QIANJU_API_KEY, // 你的千聚key
    baseURL: cfg.baseURL,
  });

  const completion = await client.chat.completions.create({
    model: cfg.model,
    messages: [{ role: 'user', content: message }],
  });

  res.json(completion.choices[0].message);
});

app.listen(3000, () => console.log('模型接入脚手架已启动'));


看明白了吗？核心变化就是请求体里的`provider`参数和对应的模型名称。你发请求时传`{ provider: 'ernie', message: '你好' }`，它就调百度ERNIE；传`{ provider: 'qwen', message: '你好' }`，秒切阿里Qwen。这一切的背后，是千聚把各平台的API统一成了OpenAI标准格式。
