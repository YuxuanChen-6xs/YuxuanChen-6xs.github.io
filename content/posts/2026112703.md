## 第三步：编写你的第一条调用代码（附逐行解释）

现在，我们来写实际的代码。创建一个新的 Python 文件，比如叫 `call_moonshot.py`，然后把下面的代码粘进去：

python
import os
from openai import OpenAI

# ⚠️ 避坑指南：API Key 千万不要硬编码在这里！
# 最好通过环境变量读取，这样既安全又方便管理。
# 在终端里设置：set OPENAI_API_KEY=你的Key
api_key = os.environ.get("MOONSHOT_API_KEY")

# 如果没有设置环境变量，也可以直接填字符串（不推荐）
# api_key = "sk-xxxxxxxxxxxxxxxxxxxxx"

client = OpenAI(
    api_key=api_key,  # 使用上面获取的 Key
    base_url="https://www.qianjuai.com/v1"  # 关键！指向[千聚api聚合站](https://www.qianjuai.com/)的中转地址
)

# 准备发送的消息
# Moonshot 的 Kimi 模型支持长上下文，所以我们可以一次性传更多内容
messages = [
    {"role": "system", "content": "你是一个知识渊博的助手，请用中文精确回答。"},
    {"role": "user", "content": "请帮我解释一下什么是“大语言模型幻觉”？"}
]

# 发起 API 调用
# ⚠️ 很多人调用失败是因为模型名字写错了！
# Moonshot 的模型名一般是 "moonshot-v1-8k" 或 "moonshot-v1-32k"
response = client.chat.completions.create(
    model="moonshot-v1-8k",  # 这里替换成你想要的 Moonshot 模型
    messages=messages,
    temperature=0.7,
    max_tokens=2048
)

# 打印输出结果
print(response.choices[0].message.content)


**代码里藏着的关键要点：**

- **`base_url`**：这是最重要的一行。如果你用的是[千聚api聚合站](https://www.qianjuai.com/)，就要改成 `https://www.qianjuai.com/v1`。
- **`model`**：Moonshot 的模型名是 `moonshot-v1-8k` 或 `moonshot-v1-32k`。如果你写成 `gpt-3.5-turbo`，那肯定报错。
- **`os.environ.get`**：推荐用环境变量读取 Key，而不是硬编码。你可以先设置环境变量，再运行脚本。

**如何设置环境变量？**

- 在 Windows 终端（CMD）里：`set MOONSHOT_API_KEY=sk-xxxxxxxxxxxxx`
- 在 PowerShell 里：`$env:MOONSHOT_API_KEY="sk-xxxxxxxxxxxxx"`
- 在 macOS/Linux 里：`export MOONSHOT_API_KEY=sk-xxxxxxxxxxxxx`

在你的终端里，定位到 `call_moonshot.py` 所在的文件夹，然后输入：

**常见报错及解决方法：**

### 1. **报错：`AuthenticationError: 401 Unauthorized`**

**原因**：API Key 无效或未正确设置。

**解决方法**：
- 检查你的 API Key 是不是完整复制了。
- 检查代码里 `api_key` 变量是否被正确赋值。
- 如果是通过环境变量读取的，确认变量名是否叫对了。

### 2. **报错：`BadRequestError: 400 参数错误`** 或 **`Model not found`**

**原因**：模型名称写错了，或者 API 地址不对。

**解决方法**：
- 在 `base_url` 里加上 `/v1` 了吗？很多新人会忘记这个。
- 模型名是不是用了 `moonshot-v1-8k` 而不是 `moonshot-v1-8k`？（注意大小写和连字符）
- 经常有人把 `moonshot` 拼成 `moonlight` 或 `moonshine`，注意拼写！

### 3. **报错：`ConnectionError: HTTPSConnectionPool`**

**原因**：网络连接失败，可能是你的网络环境无法直接访问 API 地址。

**解决方法**：
- 检查你用的中转站（[千聚api聚合站](https://www.qianjuai.com/)）是否稳定。
- 如果是连着代理，试着关闭代理或切换网络。
- 确保你的 `base_url` 地址写的是 `https://www.qianjuai.com/v1`，而不是错误的 `http://`。
