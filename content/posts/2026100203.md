## 第四步：进阶玩法——处理流式输出（Streaming）

如果你想让你的应用更有“AI实时打字”的感觉，可以启用流式输出。代码改动同样极其微小：

python
# ... 上面配置client的代码不变 ...
from openai import OpenAI

client = OpenAI(
    base_url="https://www.qianjuai.com/v1",
    api_key="sk-你的千聚API密钥"
)

# 只需在create函数里增加一个参数：stream=True
response = client.chat.completions.create(
    model="o3-mini",
    messages=[
        {"role": "user", "content": "用简单的话解释一下什么是‘递归’。"}
    ],
    max_tokens=1024,
    stream=True  # 启用流式输出
)

# 逐块处理返回的文本
for chunk in response:
    if chunk.choices[0].delta.content is not None:
        print(chunk.choices[0].delta.content, end="")  # end="" 避免每次换行


看到了吗？你只需要在传参时加上 `stream=True`，并把处理方式从 `print(response.choices[0].message.content)` 改为一个循环，就实现了流式效果。**这种改动，几乎可以说是零代码成本的体验优化。**
