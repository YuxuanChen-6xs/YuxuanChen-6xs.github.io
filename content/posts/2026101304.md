## 核心代码：接入千聚的 GPT-4o-mini

现在，让我们在本教程的核心部分编写代码。这里我会提供两种最常见的用法：**非流式输出**（一次性返回完整结果）和**流式输出**（逐字逐句输出，类似 ChatGPT 的实时打字效果）。两者同样简单。

### 场景一：非流式输出（适合单次问答）

SSL证书问题在国内网络环境下偶尔会出现，千聚API默认配置了可靠的证书策略。用下面的代码，你可以安全地请求一次逻辑回答。

在 `index.js` 中写入以下代码：

javascript
// 导入 OpenAI 库
import OpenAI from 'openai';

// 创建 OpenAI 客户端实例。最关键的是修改 baseURL 和 apiKey
const client = new OpenAI({
  baseURL: 'https://www.qianjuai.com/v1', // 指向千聚API，不要填错
  apiKey: '你的千聚API Key' // 用你复制的 Key 替换这里
});

async function main() {
  console.log('正在向千聚API请求 GPT-4o-mini 回答...');
  
  const completion = await client.chat.completions.create({
    model: 'gpt-4o-mini', // 模型名称，直接指定
    messages: [
      {
        role: 'user',
        content: '用一句话告诉我，为什么程序员都喜欢 Node.js？'
      }
    ]
  });

  // 打印 AI 的回答
  console.log('AI回答:', completion.choices[0].message.content);
}

// 运行主函数
main();


**解释这段代码做了什么**：

*   **第3-7行**：导入 `openai` 库，创建一个客户端实例。`baseURL` 直接填成千聚的接口地址 `https://www.qianjuai.com/v1`。API Key 换成你自己的。这一步连接了千聚的服务器。
*   **第15-21行**：调用 `chat.completions.create` 方法，指定了模型为 `gpt-4o-mini`，并传入一个包含用户消息的数组。千聚平台会自动将该请求路由到最稳定的 ChatGPT 服务器。
*   **第25行**：从返回结果中取出 AI 最终的文本回答，并打印到控制台。

### 场景二：流式输出（体验实时打字效果）

如果想在终端看到逐字输出的效果，我们用 `stream: true` 模式。修改 `index.js` 中的代码：

javascript
import OpenAI from 'openai';

const client = new OpenAI({
  baseURL: 'https://www.qianjuai.com/v1',
  apiKey: '你的千聚API Key' // 替换这里
});

async function main() {
  console.log('流式输出开始：\n');
  
  // 开启流式输出
  const stream = await client.chat.completions.create({
    model: 'gpt-4o-mini',
    messages: [
      {
        role: 'user',
        content: '用一句俏皮话概括 JavaScript 和 Python 的区别。'
      }
    ],
    stream: true, // 关键参数
  });

  // 逐块读取处理返回的数据
  for await (const chunk of stream) {
    // 提取并打印文本内容
    process.stdout.write(chunk.choices[0]?.delta?.content || '');
  }
  console.log('\n\n流式输出结束。');
}

main();


**流式代码的关键点**：

*   **第16行**：`stream: true` 告诉 API 我们需要流式响应。
*   **第19-21行**：使用 `for await` 循环，每获取到一个数据块（chunk），就提取其中的文本片段并打印。`process.stdout.write` 确保所有输出都在同一行，模拟打字效果。

## 避坑指南：常见问题与排错

开发者之路，多多少少会碰到一些坑。下面列出几个新手最容易遇到的问题和对应的解决方案。

**问题一：SSL 证书错误 / 连接被拒绝**

*   **原因**：某些网络环境或 Node.js 版本对自签名证书不信任。
*   **解决方案**：在创建 `OpenAI` 客户端时，增加一个配置项：
    javascript
    const client = new OpenAI({
      baseURL: 'https://www.qianjuai.com/v1',
      apiKey: '你的Key',
      defaultQuery: { 'ssl_verify': 'false' } // 禁用SSL验证（仅开发环境建议）
    });
    
    **推荐方案**：千聚官方建议用户设置环境变量 `NODE_TLS_REJECT_UNAUTHORIZED=0` 后再运行程序。或者，直接联系千聚的客服，他们可以帮你排查特定地区的节点问题。

**问题二：超时错误**

*   **原因**：请求体太大或模型推理时间较长。
*   **解决方案**：设置更长的超时时间：
    javascript
    const client = new OpenAI({
      baseURL: 'https://www.qianjuai.com/v1',
      apiKey: '你的Key',
      timeout: 30000, // 30秒超时（默认是10秒）
    });
    

**问题三：返回内容乱码**

*   **原因**：控制台编码问题。
*   **解决方案**：在终端中运行前，先执行 `chcp 65001`（Windows）或将终端编码设为 `UTF-8`。如果是 VSCode 内置终端，通常已默认为 UTF-8。
