### 第三步：核心代码

创建一个文件 `kimi_k2_safe_chat.py`，内容如下：

python
import os
import asyncio
from openai import AsyncOpenAI

# 从环境变量读取API key，绝不硬编码
api_key = os.environ.get("KIMIK2_API_KEY")
if not api_key:
    raise ValueError("请先设置环境变量 KIMIK2_API_KEY")

# 关键：base_url 必须指向千聚的中转地址
client = AsyncOpenAI(
    api_key=api_key,
    base_url="https://www.qianjuai.com/v1"
)

async def safe_chat():
    messages = [
        {"role": "system", "content": "你是KimiK2，一个聪明安全的助手。"},
        {"role": "user", "content": "请用Python写一个斐波那契数列的生成器，并解释时间复杂度。"}
    ]

    # 使用流式输出，更安全、体验更好
    stream = await client.chat.completions.create(
        model="kimi-k2",   # 此处模型名由千聚中转站映射，具体请查千聚文档
        messages=messages,
        stream=True,
        max_tokens=4096,
        temperature=0.7
    )

    print("KimiK2回复：")
    async for chunk in stream:
        if chunk.choices[0].delta.content is not None:
            print(chunk.choices[0].delta.content, end="", flush=True)
    print("\n--- 对话结束 ---")

if __name__ == "__main__":
    asyncio.run(safe_chat())


### 代码安全要点解读

- **`base_url`的指向**：这是封号风险的分水岭。指向`https://www.qianjuai.com/v1`，意味着你的请求会经过千聚的洁净中继，而不是直接撞到官方风控枪口。
- **环境变量读取**：即便你代码被传到GitHub，key也不会泄露。
- **异步流式输出**：避免一次性拉取大量数据导致资源溢出，也对网络不稳定更友好。
- **模型名称**：具体用哪个模型名（如`kimi-k2`、`kimi-k2-turbo`），需要根据千聚中转站的映射表填写。一般千聚的文档里会有明确说明，或者你在千聚的后台能看到支持的具体模型ID。
