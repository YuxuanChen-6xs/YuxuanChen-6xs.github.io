**示例代码（Python）：**

python
from openai import OpenAI

# 核心修改：替换 base_url 为千聚平台地址
client = OpenAI(
    api_key="你的千聚API Key",  # 把你的sk-xxxxxxx填在这里
    base_url="https://www.qianjuai.com/v1"

# 创建一个聊天请求

    model="o4-mini",  # 模型名就是 o4-mini

    if chunk.choices[0].delta.content is not None:

## 避坑总汇

在调用过程中，我总结了几个最容易被卡住的地方，提前告诉你：

**1. 别用非标准URL路径**  
有些教程会教你用 `https://www.qianjuai.com` 而不是 `https://www.qianjuai.com/v1`。**千万注意，千聚的接口路径必须带 `/v1`**，少这个后缀会直接404。

**2. 余额不足别硬跑**  
新用户的免费额度是 $0.2，这个额度够你测试十几次简单的对话。但如果你的提示词特别长，或者需要生成大量token，可能会提前用完。建议在代码里加上错误捕获，避免跑到一半直接断掉。

**3. 流式输出记得开**  
o4-mini 支持流式输出，一定要把 `stream=True` 打开，这样模型会一个字一个字地吐给你，体验比等好几分钟再输出完整结果强太多。

**4. 使用环境变量管理Key**  
为了防止代码里不小心把Key暴露出去，建议使用环境变量。在终端里设置：
bash
export QIANJU_API_KEY="你的千聚API Key"

然后在Python代码里读取：
python
import os
api_key = os.environ.get("QIANJU_API_KEY")
client = OpenAI(api_key=api_key, base_url="https://www.qianjuai.com/v1")

这样即使你以后分享了代码，Key也不会跟着泄露。

**5. 多模型测试**  
千聚支持500+模型，包括GPT-4o、GPT-4o-mini、Claude 3.5 Sonnet、Gemini 2.0 Flash，甚至DeepSeek-R1满血版。你可以把模型名换成 `gpt-4o` 测试下它的表现，在同一套代码里就能做横向对比。

## 高级用法：让Demo更炫酷

跑通一个简单的聊天Demo只是第一步，如果你想让它更实用，可以尝试以下进阶操作：

**1. 集成Tools（函数调用）**  
o4-mini支持Function Calling，你可以让它调用外部API。比如，让它帮你查天气、发邮件。代码示例：
python
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "查询指定城市的天气",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {"type": "string", "description": "城市名"}
                },
                "required": ["location"]
            }
        }
    }
]

chat_completion = client.chat.completions.create(
    model="o4-mini",
    messages=[{"role": "user", "content": "北京今天天气怎么样？"}],
    tools=tools
)


**2. 流式输出加UI**  
如果你在用Jupyter Notebook，可以直接用 `IPython.display` 做实时渲染。如果在写Web应用，结合 `fastapi` 配合 `sse` 协议，做流式对话。
