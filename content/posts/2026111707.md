## 环境准备与代码实现

### 1. 安装依赖

bash
npm install openai undici dotenv


`undici`是Node.js内置的HTTP客户端，完全支持连接池，性能比旧版`http/https`模块有飞跃。

### 2. 配置连接池客户端

javascript
// client.js
import { OpenAI } from 'openai';
import { Agent, setGlobalDispatcher } from 'undici';
import dotenv from 'dotenv';
dotenv.config();

// 创建连接池代理，关键参数
const agent = new Agent({
  // 连接池大小：根据你的并发需求调整
  connections: 100,
  // 连接超时（毫秒）
  connectTimeout: 10000,
  // 空闲连接保持存活时间（毫秒）
  keepAliveTimeout: 60000,
  // 每个域名最大连接数
  pipelining: 10,
});

// 设置全局dispatcher（让所有undici请求用这个池）
setGlobalDispatcher(agent);

// 用我们的连接池代理替换OpenAI SDK默认客户端
export const openai = new OpenAI({
  baseURL: 'https://www.qianjuai.com/v1',
  apiKey: process.env.QIANJUAI_API_KEY,
  // 关闭默认的fetch，使用undici
  httpAgent: agent, // 注意：这里是关键，让openai库走我们的agent
  // 替代方案：如果SDK版本不支持直接传httpAgent，可以用custom fetch
  // fetch: (url, options) => {
  //   return agent.request({ ...options, origin: url.toString() });
  // },
});


> **注意**：如果OpenAI SDK版本较新（v4.x），可能在配置上略有差异。稳妥的做法是用axios + keep-alive替代。

  }
}

## 生产环境中的注意事项

### 1. 连接池大小不要死板

不是越大越好。如果你的QPS只有10，连接池开到100反而浪费资源。根据实际流量动态调整：
javascript
// 按平均QPS * 并发请求比例来设定
const connections = Math.ceil(averageQPS * 1.5);


### 2. 超时和错误处理

连接池里某个连接卡住，会影响池内其他请求。务必加请求级别的超时：
javascript
const response = await openai.chat.completions.create({...}, {
  timeout: 30000
});
