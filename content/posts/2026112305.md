### 小白避坑指南：这三点千万别犯

我总结了几个常见的“坑”，希望能帮到你：

**坑1：API Key 直接暴露在公开仓库**

很多人在 GitHub 上分享代码时，直接把 API Key 写死在了脚本里。结果账号被他人滥用，然后就被封了。

破解方法：使用环境变量。在你的 `.env` 文件中写：

然后在代码里通过 `os.getenv` 读取，这样你的 Key 就不会泄露了。

**坑2：对模型名称不熟悉而报错**

[千聚ai中转站](https://www.qianjuai.com/)支持 500+ 模型，但 Grok4 的名字在后台可能被标记为 `grok-4`。如果你试了 `grok4`、`grok-4-beta`，可能会报错。

破解方法：直接去官方文档或后台“模型列表”里看准确名称，别靠猜。

**坑3：网络代理设置冲突**

如果你本地开了 VPN 或其他代理，千聚的调用可能会被干扰，导致连接失败。

破解方法：在代码中禁用本地代理，或者确保代理不干扰千聚的域名 `www.qianjuai.com` 和 `https://www.qianjuai.com/v1`。
