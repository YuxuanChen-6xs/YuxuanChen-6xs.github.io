### 第三步：写代码，3行核心代码召唤GPT-5nano

这一部分是最刺激的。我不想给你贴一个几百行的大块代码吓你，我只给你看最核心的“灵魂”操作。

在你的代码编辑器（比如VSCode或者记事本）里，新建一个文件，叫 `test_gpt5.py`，然后复制粘贴下面这段内容：

python
# 1. 导入库
from openai import OpenAI

# 2. 配置客户端（这两行是你要改的核心）
client = OpenAI(
    api_key="sk-你的API密钥",  # 粘贴你刚才复制的那一串
    base_url="https://www.qianjuai.com/v1"  # 这里是关键！千聚的专属地址
)

# 3. 发送请求，让GPT-5nano说话
response = client.chat.completions.create(
    model="gpt-5-nano",  # 最新最轻量的模型ID
    messages=[
        {"role": "user", "content": "你好，请问1+1等于几？"}
    ]
)

# 4. 打印结果
print(response.choices[0].message.content)


**别急着跑，先停下看看这几行代码在做什么：**

-   **第一块**：`OpenAI(...)`。这里我们告诉程序，我们用的不是官方的OpenAI，而是千聚API。所以把 `base_url` 改成了 `https://www.qianjuai.com/v1`。`api_key` 记得改成你刚复制的那一串。

-   **第三块**：`print(...)`。结果被保存在 `response` 这个变量里，我们要从中把电脑生成的文本抠出来。

### 代码跑不起来？别急，我帮你排查

小白最怕的就是报错。我这里列出最常见的三个“拦路虎”，99%的问题都能解决。

1.  **报错：`Connection Error`（连接错误）。**
    -   **原因**：网络不通或者`base_url`填错了。
    -   **解法**：检查你电脑是不是连网了？再检查下`base_url`是不是 `https://www.qianjuai.com/v1`，注意是`https`不是`http`，别忘了末尾的`/v1`。

2.  **报错：`Authentication Error`（认证错误）。**
    -   **原因**：`api_key`填错了，或者过期了。
    -   **解法**：重新登录千聚API后台，去查看你的Key是不是填对了。（建议重新复制粘贴，不要手打，手打容易漏字符）。

3.  **报错：`Model not found`（模型找不到了）。**
    -   **原因**：模型名字写错了。
    -   **解法**：先去千聚API后台的“模型列表”里，确认一下免费的或最便宜的模型ID叫什么。有时候是 `gpt-5-nano`，有时候新版本可能会加后缀。
