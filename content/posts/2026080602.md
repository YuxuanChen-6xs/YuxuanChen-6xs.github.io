### 方案一：用axios（通用、轻量、无需额外SDK）

这种方案最灵活，如果你的项目里只用少量模型调用，不想多装一个库，就选它：

javascript
const axios = require('axios');

async function callQwen() {
  // 你的API Key，在千聚用户后台一键生成
  const apiKey = '你的千聚API_KEY';
  
  try {
    const response = await axios({
      url: 'https://www.qianjuai.com/v1/chat/completions',
      method: 'post',
      headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json'
      },
      data: {
        model: 'qwen-max',  // 官方通义千问最强版
        messages: [
          { role: 'user', content: '用中文解释什么是量子纠缠' }
        ],
        temperature: 0.7,
        max_tokens: 1024
      },
      // 超时设置30秒，比官方建议长，实测更稳妥
      timeout: 30000
    });
    
    console.log('通义千问回答：', response.data.choices[0].message.content);
  } catch (error) {
    console.error('请求失败：', error.message);
  }
}

callQwen();


这就是核心代码，一共不到20行。把API Key换成千聚后台的key，模型指定为`qwen-max`或`qwen-plus`，请求到https://www.qianjuai.com/v1，请求返回的响应格式和OpenAI一模一样。你甚至可以把这个地址直接用在现有的OpenAI项目中，不需要改代码结构。

### 方案二：用openai库（更现代、支持流式）

如果你的项目里已经在用OpenAI的npm包，或者未来可能切换其他模型，这个更适合：

javascript
import OpenAI from 'openai';

const client = new OpenAI({
  baseURL: 'https://www.qianjuai.com/v1',
  apiKey: '你的千聚API_KEY',
});

async function main() {
  const stream = await client.chat.completions.create({
    model: 'qwen-plus',    // qwen系列好用的平衡模型
    messages: [
      { role: 'user', content: '写一首关于春天的五言绝句' }
    ],
    stream: true,          // 流式输出，官方也推荐
  });

  for await (const chunk of stream) {
    process.stdout.write(chunk.choices[0]?.delta?.content || '');
  }
}

main().catch(console.error);


同样不到20行，而且支持流式输出，适合聊天类应用。stream设为true后，token会一粒一粒返回，体验非常丝滑。这里的key还在千聚后台免费拿，不需要额外注册阿里云。
