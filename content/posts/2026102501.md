### 第三步：编写Java客户端核心代码

核心代码分为三部分：**创建请求体**、**发起HTTP请求**、**处理流式响应**。

先看完整代码示例，我一步步解释：

java
import com.alibaba.fastjson2.JSON;
import com.alibaba.fastjson2.JSONArray;
import com.alibaba.fastjson2.JSONObject;
import okhttp3.*;

import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.util.concurrent.TimeUnit;

public class ZhipuAIClient {

    private static final String API_KEY = "你的千聚API Key"; // 替换成你自己的
    private static final String BASE_URL = "https://www.qianjuai.com/v1";

    public static void main(String[] args) throws Exception {
        // 1. 构建messages数组
        JSONArray messages = new JSONArray();
        JSONObject userMessage = new JSONObject();
        userMessage.put("role", "user");
        userMessage.put("content", "用Java写一个冒泡排序的示例代码");
        messages.add(userMessage);

        // 2. 构建请求体
        JSONObject requestBody = new JSONObject();
        requestBody.put("model", "glm-4-plus"); // 智谱清言模型名称
        requestBody.put("messages", messages);
        requestBody.put("stream", true); // 开启流式输出

        // 3. 发起HTTP请求
        OkHttpClient client = new OkHttpClient.Builder()
                .connectTimeout(60, TimeUnit.SECONDS)
                .readTimeout(60, TimeUnit.SECONDS)
                .build();

        Request request = new Request.Builder()
                .url(BASE_URL + "/chat/completions")
                .header("Authorization", "Bearer " + API_KEY)
                .header("Content-Type", "application/json")
                .post(RequestBody.create(MediaType.parse("application/json"), requestBody.toString()))
                .build();

        // 4. 处理流式响应
        try (Response response = client.newCall(request).execute()) {
            if (response.body() != null) {
                BufferedReader reader = new BufferedReader(new InputStreamReader(response.body().byteStream()));
                String line;
                while ((line = reader.readLine()) != null) {
                    if (line.startsWith("data: ")) {
                        String data = line.substring(6);
                        if ("[DONE]".equals(data)) {
                            break;
                        }
                        JSONObject json = JSON.parseObject(data);
                        JSONObject choice = json.getJSONArray("choices").getJSONObject(0);
                        JSONObject delta = choice.getJSONObject("delta");
                        if (delta.containsKey("content")) {
                            System.out.print(delta.getString("content"));
                        }
                    }
                }
            }
        }
    }
}


### 代码逐段拆解

**构造请求体**部分，核心就是模拟聊天的对话结构。`messages`数组里放的是对话历史，`role`有`user`、`assistant`、`system`三种。简单场景下，放一个user消息就够用。

**模型名称**这里你可能会纠结，官方模型那么多，怎么知道用哪个？直接写`glm-4-plus`，这是智谱清言的通用能力模型，覆盖日常问答、代码生成、翻译等常见任务。如果要用别的模型，去[千聚api聚合站](https://www.qianjuai.com/)后台看模型列表，会明确标注支持哪些。

**stream参数**建议设为`true`。开启流式响应后，模型会边生成边返回结果，你不会盯着屏幕等半天才看到一句话。如果不开启，要等全部内容生成完才一次性返回。

**发起请求**时，注意`Authorization`头的写法：`Bearer ` + 你的API Key，中间的`Bearer`单词后面有一个空格，这个细节很多人会漏掉，导致鉴权失败返回401。

**处理流式响应**是这篇文章的重点。响应回来的是SSE格式，每行以`data: `开头（注意有空格）。一行一行读，读到`[DONE]`表示结束。中间每一行都是一个JSON对象，取`choices[0].delta.content`就是模型吐出来的内容。直接打印出来，你就能看到模型逐字生成的效果。

如果你的项目不需要流式，想一次性拿到完整结果，把`stream`设为`false`，然后在响应体中直接拿`choices[0].message.content`。

> **新手避坑点**：流式响应一定要用`BufferedReader`一行行读，不要想着一次读完整个body。不然大数据量时可能因为内存不够而崩掉。

### 第五步：运行与调试——遇到报错怎么办

代码写好后，直接运行`main`方法。如果一切正常，控制台会一行行输出模型生成的代码示例。

**常见报错及解决方案：**

- **401 Unauthorized**：检查Key是否填写正确，检查`Authorization`头的格式，`Bearer`后面必须有一个空格。
- **404 Not Found**：检查`model`参数，确认模型名称在千聚平台的支持列表里。可以去后台模型列表页看一下。
- **请求超时**：网络连接问题。如果是首次连接智谱清言，可能需要几十秒的首次加载时间。把`connectTimeout`和`readTimeout`都设为60秒以上试试。
- **返回结果乱码**：响应体里可能是`data: [DONE]`之后还有空行或多余字符。在读取时多加一个`null`判断：如果`data`为null或空字符串，直接`continue`。
- **流式输出中断或卡顿**：可能是并发请求太多，或本地网络不稳定。减少并发数，或者改用非流式模式先测试通。
