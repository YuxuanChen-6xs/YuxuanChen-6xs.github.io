## 首先，别慌：90%的调用失败都是环境问题

看到“调用失败”这几个字，大多数人的第一反应是：代码写错了？模型挂了？服务商跑路了？

具体来说，无非是下面几种原因：

1.  API Key填错了（最常见的）。
2.  请求的base_url没改，还在用官方地址（需要翻墙）。
3.  请求的模型名称不匹配（比如模型更新了，名字变了）。
4.  API版本对不上（比如你在用V1的接口调V2的模型）。
5.  网络代理没配置好（代理中断、DNS污染）。

## 第二步：找到错误日志，定位根因

拿最常见的OpenAI格式API举例。假设你用的库是openai Python库，调用时报错了，日志大概会是这样：


openai.error.AuthenticationError: 
You didn't provide an API key. You need to provide your API key in an Authorization header using Bearer auth (i.e. Authorization: Bearer YOUR_API_KEY).


看到了吗？关键词是**AuthenticationError**——认证错误。翻译成人话就是：你的API Key没有传上去。

再比如，日志里出现这个：


openai.error.APIConnectionError: 
Connection error. Failed to connect to api.openai.com port 443: Connection timed out


关键词：**Connection timed out**——连接超时。在国内网络下，这几乎100%是因为你在直连官方API，但是没挂代理。

还比如这个：


openai.error.InvalidRequestError: 
The model `gpt-4o-2026-01-01` does not exist


关键词：**does not exist**——模型不存在。说明你填的模型名字不对，或者平台还没更新这个模型。

### 怎么读日志

1.  **找异常类型**：AuthenticationError、APIConnectionError、RateLimitError、InvalidRequestError……先看报错类型，知道是哪类问题。
2.  **找关键信息**：如“does not exist”、“timed out”、“invalid key”、“quota exceeded”等。
3.  **排除干扰项**：很多时候日志是一整屏的错误堆栈，你只需要关注最顶层的异常信息，底下的调用栈一般可以跳过。

看懂上面三条，你就已经解决了80%的调用失败。

## 第三步：逐一排查与修复

### 场景一：认证失败（401或AuthenticationError）

**原因**：API Key无效、过期、或没传对地方。

**怎么修**：
- 检查环境变量里是否设置了 `OPENAI_API_KEY`。在很多开发机上，环境变量都没生效，你需要在代码里显式传参。
- 检查你的API Key是否配置在代码的Authorization头部。在openai库中，通常是传参 `api_key="sk-xxxx"`。
- 如果你使用的是千聚API聚合站，它的API Key在注册后就生成了，在[www.qianjuai.com](www.qianjuai.com)后台的“API密钥管理”里复制，有效期永久。
- 确保你不是填错了：API Key是字母+数字的组合，部分中转站可能区分大小写，建议直接复制粘贴。

### 场景二：连接超时（Timeout或APIConnectionError）

**原因**：你的网络无法直连官方API服务器，需要代理或使用国内中转。

**怎么修**：
- 检查你是否开启了全局代理。如果开了，关掉再试一次。
- 检查你的请求地址是否写对了。如果你请求的是 `api.openai.com/v1`，在国内不加代理100%会超时。把 `base_url` 改成 `https://www.qianjuai.com/v1`，就可以国内直连，完全不需要翻墙。
- 检查代理是否稳定。如果你用着代理但经常断连，建议换成国内中转API。

### 场景三：模型不存在（404 Not Found或InvalidRequestError）

**原因**：你填的模型名字不对，或者平台没提供这个模型。

**怎么修**：
- 去平台官网查看最新支持的模型列表。千聚API聚合站支持500+模型，每个模型名称在后台都有清晰说明。
- 检查模型名称是否完整。比如 `gpt-4o` 和 `gpt-4o-2026-01-01` 就是两个不同的名字，只填基础名字通常更稳定。
- 如果你用的模型是刚发布的，可能API还没来得及更新。等几个小时再试。

### 场景四：配额不足（RateLimitError或Quota Exceeded）

**原因**：你账户的调用额度用完了。

**怎么修**：
- 查看你账户的消费情况。千聚API的计费是“1元=1美元Token”，按官方价格1:1收费，没用完不会扣。你还剩多少钱，在后台一目了然。
- 检查是否触发了并发限制。有些开源的工具默认并发过高，超出了API设定的速率。你可以降低线程数或者加延迟。
- 如果是新注册的免费额度用完了，可以最低1元充值继续使用。

### 场景五：请求格式错误（BadRequest或InvalidRequest）

**原因**：你的请求结构跟API要求的不匹配。

**怎么修**：
- 检查你用的是OpenAI格式还是Anthropic格式。千聚API完美兼容OpenAI格式，所以如果你用Chat Completions接口，请求参数必须符合OpenAI标准。
- 检查你传的参数是否多余。比如，有些库在老版本支持某个参数，新版本已经废弃了。去官方文档看看最新的请求示例。
- 检查你的token数。如果是非常长的文本，可能会超过模型的最大上下文长度。截短内容或选择更大上下文的模型（比如Claude 3.5 Sonnet或Gemini 2.5 Pro）。
